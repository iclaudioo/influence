# Telegram Content Bot — Design Document

**Date:** 2026-03-07
**Status:** Approved

## Overview

A Telegram bot that takes a topic message, generates blog and/or LinkedIn content in the Influence Circle brand voice via Claude API, saves drafts to Supabase, and sends previews back in Telegram.

## Architecture

Two Supabase edge functions handle the flow:

```
You (Telegram) → Telegram API → telegram-webhook (edge function)
                                        ↓
                                  1. Validate sender (chat ID whitelist)
                                  2. Parse message → detect format (blog/linkedin/both)
                                  3. Reply "Bezig met genereren..."
                                  4. Invoke generate-content (fire-and-forget)
                                  5. Return 200 immediately

                                generate-content (edge function)
                                        ↓
                                  1. Call Claude API → generate content in brand voice
                                  2. Save draft to blog_posts / linkedin_posts
                                  3. Reply with preview via Telegram API
```

### Why Two Functions

Telegram requires fast webhook responses. The `telegram-webhook` function acknowledges within 1-2 seconds. The `generate-content` function handles LLM calls (15-30s) without timeout pressure.

## Format Detection

Simple keyword matching from Dutch messages:

- Contains "blog" / "artikel" / "post" → blog post
- Contains "linkedin" → LinkedIn post
- Contains both or neither → generates both formats

## Content Generation

### Claude API

- **Model:** claude-sonnet-4-6
- **SDK:** Anthropic SDK for Deno
- **Language:** Dutch only

### Brand Voice System Prompt

Built from existing site copy:

- **Persona:** Claudio Swijsen, founder Influence Circle, Reputation Architect
- **Tone:** Strategisch, direct, anti-hype, geen buzzwords, geen AI-slop
- **Frameworks:** Reputation Architecture Model (Zichtbaar → Herkenbaar → Vertrouwd → Onvermijdelijk), 4 service lines (Labs, Circle, Studio, Academy)
- **Style:** Kort en krachtig, geen uitroeptekens, actieve zinnen, concrete voorbeelden boven abstracte beloften

### Output Per Format

| Format | Fields |
|--------|--------|
| **Blog** | `title`, `excerpt` (2 zinnen), `content` (TipTap-compatible JSON), `tags[]`, `service_line` |
| **LinkedIn** | `text` (max ~1300 chars), `hook` (eerste zin), `hashtags[]` |

## Data Model

### Blog Posts (existing table)

Drafts saved to `blog_posts` with:

- `status: 'draft'`
- `title_nl`, `excerpt_nl`, `content_nl` filled
- `title_en`, `excerpt_en`, `content_en` left empty
- `service_line` auto-detected by Claude
- `tags` generated by Claude
- `cover_image_url` left null (added later via NanoBanana)
- `author_id` set to Claudio's team member ID

### LinkedIn Posts (new table)

```sql
linkedin_posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content TEXT NOT NULL,
  hook TEXT,
  hashtags TEXT[],
  topic TEXT,
  status TEXT DEFAULT 'draft',
  created_at TIMESTAMPTZ DEFAULT now(),
  published_at TIMESTAMPTZ
)
```

## Image Generation

Not automated in the bot flow. Cover images are generated separately via NanoBanana MCP tool when reviewing drafts in the admin dashboard. This keeps the edge function simple and avoids timeout issues.

## Telegram Reply Format

### Blog Post

```
BLOG DRAFT OPGESLAGEN

Titel: [title]
Tags: [tags]
Service line: [service_line]

Preview:
[first ~200 chars of content]

→ Bewerk in admin: influence.be/admin/blog/[id]/edit
```

### LinkedIn Post

```
LINKEDIN DRAFT OPGESLAGEN

Hook: [first line]

[full text]

[hashtags]
```

### Error

```
Generatie mislukt. Probeer opnieuw.
```

## Security

- Chat ID whitelist: only messages from `TELEGRAM_CHAT_ID` env var are processed
- All other messages are silently ignored
- Edge functions use existing Supabase service role key for DB access

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `TELEGRAM_BOT_TOKEN` | Bot API token from @BotFather |
| `TELEGRAM_CHAT_ID` | Your personal chat ID (security) |
| `ANTHROPIC_API_KEY` | Claude API for content generation |

Existing Supabase vars (`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`) are already available in edge functions.
